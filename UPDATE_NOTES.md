## 🆕 新功能

### 自动检测并释放端口

启动脚本现在会自动检测端口占用情况：

1. **检测后端端口 8000**
   - 如果被占用，自动停止占用进程
   - 等待端口释放后继续启动

2. **检测前端端口 3000**
   - 如果被占用，自动停止占用进程
   - 等待端口释放后继续启动

### 工作流程

```
启动脚本
    ↓
检查后端环境
    ↓
检查前端依赖
    ↓
创建日志目录
    ↓
检查后端端口 8000
    ↓ (如果被占用)
停止占用进程
    ↓
检查前端端口 3000
    ↓ (如果被占用)
停止占用进程
    ↓
启动后端服务
    ↓
启动前端服务
    ↓
完成
```

## 🐛 错误处理

### 无法释放端口

如果端口被占用且无法释放，脚本会：

1. 显示错误信息
2. 停止启动流程
3. 不启动任何服务

```
❌ 无法启动后端，端口 8000 被占用且无法释放
```

**解决方法**：
```bash
# 手动查找并停止进程
lsof -ti :8000 | xargs kill -9
```

## 💡 注意事项

1. **自动停止进程**：脚本会使用 `kill -9` 强制停止占用端口的进程
2. **等待时间**：端口释放后会等待 2 秒确保完全释放
3. **Ctrl+C 安全**：使用 Ctrl+C 可以安全停止所有服务
4. **权限要求**：脚本需要权限停止其他用户的进程，可能需要 sudo

## 📞 常见问题

### Q: 如何防止误停止其他服务？

A: 脚本会精确匹配端口号，不会误停止其他服务。

### Q: 脚本能否停止其他用户的进程？

A: 可以，但需要 root 权限。如果权限不足，会显示错误。

### Q: 如何查看哪些进程被停止了？

A: 脚本会显示被停止的 PID，例如：
```
正在停止进程: 12345, 12346
```
